Title: JeopardyLLM: Adventures in Retrieval-Augmented Generation

 

-         Job Bangayan, Jerry Gonzalez, Sean Huang, Vineeta Kumar, Tianyu Wang

 

Abstract

 

***

 

 Introduction

 

-         Jeopardy itself:

-          Within the trivia world, no question-and-answer format is more pervasive and enduring than Jeopardy's. This project aims to develop a model that allows for the answering questions based on a model trained on previous Jeopardy questions (over 470,000 clues over close to 30 seasons) by way of contextual embeddings and retrieval tasks.

-         Information retrieval:

-          The implications of information retrieval are vast making. In fact, information retrieval is a core principle in natural language processing and this project helps the project team develop skills to become experts in this space. Being able to answer user queries is a foundational part of question answering systems. These skills can be practiced on a set using Jeopardy questions.

-          We have not found any projects where a language model is trained on Jeopardy questions. (making this a compelling and impactful problem to solve, big opportunity)

-          We do not expect that results to be spectacular for all use cases. Therein lies some potential interesting analysis that can be performed by way of language and subject analysis.

-          Our JeopardyLLM offers an unparalleled experience for trivia enthusiasts and knowledge seekers by harnessing the power of advanced natural language processing.

-          JeopardyLLM is powered by cutting-edge NLP technology that ensures precise and accurate responses.

-          JeopardyLLM is category agnostic in that it handles questions from a wide range of topics and categories found in Jeopardy. However, it is expected to perform better on certain categories than others.

-          We want the user to have a fun experience by providing a simple and intuitive interface for submitting questions. This ensures that users can effortlessly engage with the platform and enjoy the experience.

-          We will optimize for real-time responsiveness where users receive answers promptly creating a dynamic experience.

-          How to quantify impact, market opportunity: There may be some impact in the trivia world. We aim to create a website where trivia enthusiasts can experiment with questions for free and train themselves on trivia. Generalizability studies on non-Jeopardy questions may be performed and analyzed for accuracy.

-          Define the primary use case validated by this target user/customer: “Can I answer this question using solely information from older Jeopardy questions?”

 

Background

-          Market Size: Jeopardy is a very popular game show, averaging 9.2 million total viewers across the 2021-2022 season. There are many more interested in trivia and similar games throughout the country and the world at large.

-          We have not found any projects where a language model is trained on Jeopardy questions.

-          Some assumptions we will make about the problem/opportunity: That trivia enthusiasts and other curious users will be interested in knowing whether a strength in trivia and fact information applies to knowledge and interests outside of trivia and towards answering more novel questions.

-          Target user/customer segments: Trivia enthusiasts, prospective game show contestants who want to consolidate their knowledge base, anybody who is curious. Assumption that 100% accuracy, that the users are merely curious. That users would structure questions in a way that is answerable.

-          Major players and main vendors in the space, existing solutions: ChatGPT, other LLMs that can be fine-tuned to answer questions. Many of these LLMs may be found on Huggingface. Google Cloud AI, AWS, Rasa (table from project plan)

-          Answering Jeopardy-style questions using algorithms became especially popular when IBM Watson defeated co-game show legends Ken Jennings and Brad Rutter in 2013 via a simple stochastic model (Tesauro, 2013).

-          Some newer language transformer models, such as BARD and ChatGPT, perform this task with the accuracy of a human expert. Moreover, they demonstrate higher accuracy than Watson in doing so. (O’Leary, 2023) 

-          Jeopardy questions can be challenging to parse, given the ambiguity within the questions. Practicing and gaining experience with answering questions is helpful for prospective contestants. In addition, some question topics are more popular than others (history over sports, for instance). Some studies explore trends while analyzing the questions themselves.

-          Similar question and answering tasks have used SQuAD as a context

-          Rajpurkar, P., Zhang, J., Lopyrev, K., & Liang, P. (2016). Squad: 100,000+ questions for machine comprehension of text. arXiv preprint arXiv:1606.05250.

-          Labeling may be necessary if a dataset is ambiguous as Jeopardy’s dataset

 

Methods

-          Questions and answers are found on the Jeopardy Clue Dataset curated online2 by github user jwolle1 (https://github.com/jwolle1/jeopardy_clue_dataset). The dataset contains 473,067 Jeopardy clues from Season 1 up to Season 39 which ended in July 2023. Notedly, the site states that it does not contain every clue that has appeared on the show and the data source preferred not to be credited. Each of the questions were uploaded for analysis. 50 different clues were generated using a random number generator. The number ’50’ was chosen because that it is number of questions on the Jeopardy Anytime test. In addition, creating contexts for questions from a large dataset can be time and resource heavy, and opting for 50 seemed clean.

-          Public dataset. Data is relatively clean already, questions and answers

-          Regarding the user journey and UI/UX for this data product: The user merely needs to interact with the website. The homepage is clean and encouraging. They would not need to sign up or register. They simply need to type in questions. There will be instructions. Feedback will be in the form of short answers. The users themselves will be determined whether they are correct or not. There will be an area to solicit feedback.  Payment is not expected.

-          Methodologies used for initial EDA - JSON file, via ChatGPT 4.0, look at categories, questions, and answers

-          Comparing Models from Huggingface Leaderboard

-          Retrieval Augmented Generation

-          We will evaluate the accuracy of the model’s responses by comparing them against the correct answers in the Jeopardy dataset.

-          Will collect user feedback regarding the relevance and correctness of the answers provided.

-          Will measure response time to ensure that user expectations are met for a quick and seamless experience.

-          Flan-T5 which has been trained on question-answer pairs and better follow directions. Flan-T5 can incorporate contexts. Results will be compared with an pretrained Flan-T5 as well. 

-          The results from Flan-T5 answering task will be compared to the correct answers. We will provide an exact match in the Jeopardy vein meaning a few things, however similar to the game show, spelling does not matter unless a syllable is added. In addition, last names are sufficient for proper names. Any grammatically correct question with the correct answer within it counts as well. Each contestant will display their own individual match metric however we will also have an overall metric (where at least any one of the three contestants is correct – this is similar to the game show where only one needs to buzz in correctly for points to be awarded. However Jaccard similarities rely on exact matches, rather than nuanced matches by Jeopardy standards.

 

Results

-          The minimal viable product is a basic website that allows users to input a question and receive an answer that is obtained from a predefined set of Jeopardy questions. The goal is to test the fundamental assumptions about the ability of the language model to understand and respond appropriately to a range of Jeopardy-style questions. Main features - What are the main features and why? Will have a simple question interface where users can submit questions. Currently looking at using a text box. There will be a section to display the generated answer. Will need to integrate a Jeopardy dataset that contains a diverse set of Jeopardy questions and answers.

-          More about website: instructions, tip sheet for how to prompt engineer. There will be an area on the website to solicit feedback for continued improvements.

 

Discussion

-          Our MVP demonstrates the ability to generate answers based solely on older Jeopardy questions. It is essentially a test to see whether an extensive trivia and Jeopardy base is enough to justify questions that are not from game shows. Existing solutions are expectedly much more powerful and draw on larger sources.

-          Improving the website, other potential improvements

-          Value delivered - The primary value is delivering a quick and accurate response to Jeopardy-style questions.  Users will be able to challenge the system’s ability to comprehend and answer questions.

 

Conclusion

-          The project demonstrates moderate success at answering newer Jeopardy questions using older ones.
